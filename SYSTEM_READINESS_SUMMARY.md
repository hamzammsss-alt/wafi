## 🔗 آلية الربط في السندات

### أ. على مستوى رأس السند (Header)
```javascript
{
  id: "REC-001",
  type: "RECEIPT",              // نوع السند
  partner_id: "CUST-123",       // معرّف العميل
  linked_account_id: "114001",  // حساب الربط
  total_amount: 5000,
  created_at: "2026-04-11"
}
```

### ب. على مستوى أسطر السند (Lines)
```javascript
{
  id: "RECV-LINE-001",
  journal_entry_id: "JE-001",
  account_id: "112",            // حساب النقد
  debit: 5000,
  sub_account_id: "CUST-123",   // معرّف الكيان الفرعي
  invoice_ref: "INV-2024-001",  // المرجع
  customer_id: "CUST-123",      // ربط إضافي
  tax_ref: "TAX-REF-001",       // مرجع ضريبي
  is_returned: false
}
```

## 📄 السندات المدعومة

### 1. **سند القبض** (Receipt Voucher)
```
الكود: RECEIPT
الاستخدام: تسجيل دفعات العملاء
الربط: customer_id → linked_account_id → invoice_ref
مثال:
  - عميل يدفع 5000 ريال
  - السند يربط بحساب العميل (114xxx)
  - يسجل في حساب النقد/البنك (111x/112x)
```

### 2. **سند الصرف** (Payment Voucher)
```
الكود: PAYMENT
الاستخدام: تسجيل دفعات الموردين
الربط: supplier_id → linked_account_id → po_number
مثال:
  - دفع لمورد 10000 ريال
  - السند يربط بحساب المورد (211xxx)
  - يصدر من حساب البنك (112x)
```

### 3. **سند القيد اليومي** (Journal Voucher)
```
الكود: JOURNAL
الاستخدام: قيود محاسبية عامة
الربط: حسابات متعددة + مراجع اختيارية
مثال:
  - تحويل بين حسابات
  - مصاريف دورية
  - استهلاك الأصول
```

### 4. **سند الرواتب** (Payroll Voucher)
```
الكود: PAYROLL
الاستخدام: تسجيل رواتب الموظفين
الربط: employee_id → salary_account → deduction_accounts
مثال:
  - رواتب الموظفين
  - البدلات (سكن، نقل)
  - الخصومات (تأمين، قروض)
```
